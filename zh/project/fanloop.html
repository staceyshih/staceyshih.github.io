FanLoop 潮流影音 App

Overview

在 3 個月內，從 0 到 1 設計「長、短影音並存」的潮流影音平台 App — 規劃並執行商業概念驗證研究、UIUX 設計，為 CATCHPLAY 拓展短影音市場版圖奠定基礎。

*此 App 目前為 Soft Launch 階段，已累積 100+ 次下載、500 小時影音內容，並與 50+ 品牌、創作者接洽合作中。

Goal

商業目標
CATCHPLAY 集團除了有 OTT 影音平台（CATCHPLAY+)，同時兼有電影發行、影劇製作...等業務。為了達到最佳商業綜效，決心拓展影音市場版圖，在原有的「長影音」市場之外，開拓「免費短影音」的市場。

短期來說，能增加自製或代理的長影音內容曝光與傳播效率、賺取廣告收入。長期亦可作為進入印尼、台灣等東南亞國家的短影音創作市場的基石，以更貼近影視產業需求的客製化功能做為誘因，吸引追求高品質的內容創作者合作獨家上架內容，增加平台流量。


專案目標
  1. 因應商業需求，設計一款可「兼容」長、短影音觀看的 App，提供客製化導流功能。
  2. 「短影音」是團隊不熟悉的題目，團隊共識在進入開發前，需先進行使用者研究以驗證商業概念，並依研究結果進行設計優化。

Background

現今人們觀影時間更加零碎化，團隊觀察到人們「觀看長時間影片」的意願降低，反而是「觀看可中斷影片」的時間增加，希望設計一款服務，滿足人們把影音內容當做點心的這種需求，同時對原有平台的長影片內容進行更好的導流。

Research

研究目的

由於此 App 需求是由商業端發起，研究重點聚焦於 提升使用意願 與 驗證設計假設

  1. 了解目標使用者對「短影音 App」的認知與使用契機
  2. 此產品中需要涵蓋哪些功能？什麼部分最優先需要被實踐？
  3. 搜集更多資訊幫助制定設計決策

並依此三個重點進行以下的研究規劃。

研究對象

依照 FanLoop 的目標受眾，鎖定年齡區間 25-35 歲，且符合以下兩種條件的族群：

  1. 熟悉 CATCHPLAY+ 且喜愛其影音內容的現有會員
  2. 有短影音使用經驗的非會員

研究方法

為了達到快速的概念測試、定義 MVP 功能範圍，我們在研究過程中應用以下兩種研究方法:

  1. 競品研究

    2. 在訪談前積極比較多家短影音 App 並快速實作出符合商業團隊提案之 App Prototype

  2. 1 對 1 使用者訪談

    3. 在訪談過程中，了解使用者們對短影音 App 的想像和期望。並在第二階段分享目前的 prototype 驗證對於 UI 設計的期望與建議。

  3. 狩野分析

    4. 取狩野分析的研究特長 - 了解在功能藍圖中，哪些功能最具有吸引力？找出短影音哪些是基本功能、魅力功能以及期望功能

研究流程

  1. 訪談目的說明
  2. 短影音 App 的使用習慣與使用動機、體驗分享
  3. FanLoop App 概念說明 & 操作
  4. KANO Research 問卷，同時進行簡單訪談
  5. 整體回饋分享

由於時間較倉促，為了最大化可訪談的對象，我們選擇以「遠端訪談」的方式進行。除了用簡報說明，亦提供可操作的 Prototype 網址，讓使用者可以一邊操作、一邊釐清不理解的地方。幫助使用者在研究中可以更好地理解，進而得到更加準確地回答和資訊、增加概念驗證的有效性。

ps. 為了更加擬真，Prototype 中的影音內容亦依照當下的話題和編輯精選題材進行優化。

Insights

1. 短影音閱聽動機：希望「有意義地」浪費時間
使用者選擇傾向被動，太多選項易造成選擇障礙。80% 的受訪者表示想要更沈浸直接就開始看，且若不喜歡看就滑掉即可。
2. 短影音作為「索引」，並非替代長影音的觀看，而是資訊篩選的過程
使用者大多重視時間的運用，即便在觀影過程中也會不斷地進行「重要片段」的篩選。例如：採用倍速播放，或跳過不重要的片段。偏好先觀看短影音進行篩選，再決定是否觀看長影音。
3. 內容策略具關鍵影響
「獨家內容」為主要下載誘因，「社群互動」則非關鍵，因為已有慣用的平台。有趣的是，觀察到使用者們對短影音的「原創版權」相當重視。有部分使用者提及：因為在 Instagram 看到從 TikTok 搬運過來的影片片段，進而去下載 Tiktok 去支持原始創作者或完整影片內容。
4. 「觀看影片」相關的功能最應優先開發
許多影片功能已是必備基本盤，例如：調整播放速度、自由拖曳時間軸...等，沒有做好會讓 user 失去信任。其餘的例如：個性化推薦機制、留言、收藏...等等功能，可說是較為次要。另外，使用者大多數不需要製作個人影片的需求，對他們來說在這裡只想看到較高品質的影音內容。
（待改！！！）整體來說，兩群使用者差異不大：原先較擔心熟悉 CP 的現有用戶會對短影音較為反感或不熟悉，但新手跟熟悉在對此 App 的功能偏好上沒有太多差別，代表內容偏好對此 App 的接受度較無太大影響

Design Strategies

1. 打開 App 立即進入觀看情境，減少選擇障礙

首頁以較大的版面 混合長、短影音內容，進入偵測範圍內便自動開始播放影片。

2. 結合直覺的操作手勢，由使用者掌握欲觀看內容的關聯性

為了達到有意義的探索，讓使用者在瀏覽時可以控制探索的廣度和深度，避免完全隨機的內容。
綜合現行常見的滑動手勢，不同方向的滑動代表不同關聯度的切換，並將此實踐在我們的 App 上，讓使用者不論怎麼無腦、怎麼滑都能快速接觸到有趣的內容：

3. 總是有下一步行動

使用者的瀏覽不會在影片看完後結束，我們提供 客製化的外站導流按鈕，透過短影音讓大家能外連到相關連結，例如：相關長影音的觀看管道、該影劇的官方網站、媒體報導原文...等等功能。另外，使用者亦可主動探索「創作者頻道頁」或是點擊 Hashtag 看更多相關影片。

Outcome

這次研究不僅影響了 UI 設計，也對內容上架策略與商業決策帶來貢獻。針對產品團隊的貢獻則主要使以下幾點：

  1. 幫助團隊聚焦 MVP 開發，縮短決策時間

    2. 因為當時開發資源亦不足，需要選擇先上最重要的功能。透過此研究節省了團隊探索的時間，可以快速達到 MVP 定義的共識。

  2. 加強短影音操作體驗，讓 UI 設計更貼近使用者操作習慣

    3. 如上述研究發現，優先加強影音觀看、操作的 UI 讓產品正式推出時，使用者們可以更容易上手操作。

  3. 更有效率的工作流程

    4. 因為是一個比較新的產品概念，同時是團隊內部不熟悉的題目。當時為了研究製作的 短影音 prototyping，也廣泛應用於簡報、跨部門討論、以及給內容、編輯團隊與外部內容提供者的合作提案簡報中。

Learning & Next Step

需考量終端使用者、內容創作者真實的需求

針對不熟悉的領域，除了研究 TA，也需要在設計過程中去同理內容創作者，還有團隊內部的內容上架人員，目前的設計是否會增加維運的成本？是否有必要？

產品概念的驗證仍有所限制

不論是否為現有用戶，是否真的會有意願使用此產品仍難以此研究進行評判，但起碼透過研究得知了「可獲得認可的產品價值」、「現有的設計概念如何優化以更符合使用需求」

遠端訪談無法在使用者旁觀察操作、提供指引，Prototype 產生的效果仍有所打折。透過正式訪談前的 Pilot Test，及時優化、更新研究素材，可以更有效率地收穫精確的回答。

當時還有遇到：KANO 訪談的功能排序無法說服所有人的問題，後續團隊進行檢討時發現：研究者需要與所有「有權力決策開發方向」的 Stakeholders 確認納入研究排序的功能項目和表達方式，並達到共識，否則最後開發順序較難完全按照研究結果進行。
